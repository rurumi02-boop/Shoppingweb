# 修復註冊功能錯誤

## 問題說明

錯誤訊息：
```
OperationalError: no such table: myapp_user
```

**原因**：
- Django 的 User 模型需要資料庫表
- 遷移尚未執行，所以表不存在
- 註冊功能使用 `User.objects.create_user()` 需要資料庫表

## 已解決

✅ **已執行遷移**：`py -3.11 manage.py migrate`

所有必要的資料庫表已創建：
- ✅ contenttypes
- ✅ auth（使用者認證）
- ✅ myapp（自訂 User 模型）
- ✅ admin
- ✅ sessions

## 當前架構

### 混合架構

您的專案現在使用**混合架構**：

1. **Django User 模型**（SQLite 資料庫）
   - 用於使用者認證（登入/登出）
   - 儲存在 SQLite 資料庫中
   - 這是 Django 認證系統的基礎

2. **Firebase Firestore**（Firebase 資料庫）
   - 用於商品、訂單、我的最愛等業務資料
   - 儲存在 Firebase 中

### 為什麼需要 Django User 表？

- Django 的認證系統（`authenticate()`, `login()`, `logout()`）需要資料庫表
- Session 管理需要資料庫表
- 即使使用 Firebase，Django 的認證仍然需要這些表

## 註冊功能說明
