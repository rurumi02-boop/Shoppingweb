# 🚀 推送專案到 GitHub - 完整解決方案

## 當前狀況
系統未安裝 Git，需要先安裝才能推送。

---

## 方案一：安裝 Git 後推送（推薦）⭐

### 步驟 1：安裝 Git

1. **下載 Git**：
   - 前往：https://git-scm.com/download/win
   - 點擊下載按鈕（會自動選擇適合的版本）

2. **安裝 Git**：
   - 執行下載的安裝程式
   - **重要**：在安裝過程中，找到「Adjusting your PATH environment」選項
   - 選擇「**Git from the command line and also from 3rd-party software**」
   - 其他選項保持預設即可
   - 完成安裝

3. **驗證安裝**：
   - 關閉所有命令提示字元和 PowerShell 視窗
   - 開啟新的命令提示字元或 PowerShell
   - 執行：`git --version`
   - 如果顯示版本號，表示安裝成功

### 步驟 2：執行推送

安裝 Git 後，在專案目錄（`C:\CampingData`）執行：

**方法 A：使用自動化腳本**
```bash
快速推送.bat
```

**方法 B：手動執行命令**
```bash
git init
git remote add origin https://YOUR_GITHUB_TOKEN_HERE@github.com/rurumi02-boop/CampingData.git
git add .
git commit -m "Initial commit: CampingData Django e-commerce project"
git branch -M main
git push -u origin main
```

## 方案二：使用 GitHub Desktop（最簡單）🖥️

如果不想安裝 Git 命令列工具，可以使用 GitHub Desktop：

### 步驟 1：下載並安裝 GitHub Desktop

1. 前往：https://desktop.github.com/
2. 下載並安裝 GitHub Desktop
3. 使用 GitHub 帳號登入

### 步驟 2：推送專案

1. 開啟 GitHub Desktop
2. 點擊 **File** → **Add Local Repository**
3. 選擇專案目錄：`C:\CampingData`
4. 如果提示需要初始化，點擊 **create a repository**
5. 點擊 **Publish repository** 按鈕
6. 輸入倉庫名稱：`CampingData`
7. 選擇是否公開（Public/Private）
8. 點擊 **Publish Repository**

**注意**：使用 GitHub Desktop 時，Token 會自動處理，不需要手動輸入。

---

## 方案三：使用網頁上傳（臨時方案）🌐

如果急需上傳且不想安裝任何工具：

### 步驟 1：創建 GitHub 倉庫

1. 前往：https://github.com/new
2. 倉庫名稱：`CampingData`
3. 選擇 Public 或 Private
4. **不要**勾選「Initialize this repository with a README」
5. 點擊 **Create repository**

### 步驟 2：上傳文件

1. 在 GitHub 倉庫頁面，點擊 **uploading an existing file**
2. 將專案文件拖放到瀏覽器視窗
3. 在下方輸入提交訊息：`Initial commit: CampingData Django e-commerce project`
4. 點擊 **Commit changes**

**注意**：此方法需要手動上傳每個文件，較為繁瑣。

---

## 方案四：使用 VS Code（如果已安裝）💻

如果已安裝 Visual Studio Code：

1. 開啟 VS Code
2. 開啟專案資料夾：`C:\CampingData`
3. 點擊左側的「Source Control」圖示（或按 `Ctrl+Shift+G`）
4. 點擊「Initialize Repository」
5. 點擊「...」選單 → 「Remote」 → 「Add Remote」
6. 輸入名稱：`origin`
7. 輸入 URL：`https://github.com/rurumi02-boop/CampingData.git`
8. 點擊「✓」提交所有更改
9. 點擊「...」選單 → 「Push」 → 選擇 `origin/main`

**注意**：VS Code 需要安裝 Git 擴充功能，但會自動處理認證。

## 📋 推薦順序

1. **方案一**（安裝 Git）- 最靈活，適合長期開發

3. **方案四**（VS Code）- 如果已在使用 VS Code
4. **方案三**（網頁上傳）- 僅作為臨時方案
